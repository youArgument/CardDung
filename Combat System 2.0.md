# Руководство по реализации системы характеристик для карточного рогалика

## Цель

Сделать так, чтобы любой класс мог использовать любую карту, но эффективность карты зависела от характеристик персонажа.

Игрок никогда не должен получать полностью бесполезную карту. Вместо запрета на использование применяется штраф эффективности.

Основная идея:

* Любая карта может быть разыграна любым персонажем.
* Недостаток нужной характеристики уменьшает эффективность карты.
* Достаточная характеристика раскрывает карту на 100%.
* Высокая характеристика дополнительно усиливает карту.

---

# Основные характеристики

```
STR (Strength)
Сила.
Отвечает за физический урон.

AGI (Agility)
Ловкость.
Отвечает за скорость, критические удары и мобильность.

INT (Intelligence)
Интеллект.
Отвечает за магический урон.

WIL (Willpower)
Сила воли.
Отвечает за баффы, лечение, контроль и длительность эффектов.

VIT (Vitality)
Выносливость.
Отвечает за здоровье, защиту и регенерацию.
```

---

# Структура карты

Каждая карта должна содержать следующие параметры.

```csharp
BaseDamage

MainStat

RequiredStat

Scaling

DamageType

Tags
```

Пример:

```json
{
    "Name": "Fireball",
    "BaseDamage": 6,
    "MainStat": "INT",
    "RequiredStat": 8,
    "Scaling": 0.30,
    "DamageType": "Magic",
    "Tags": ["Fire", "AOE"]
}
```

Для физической карты:

```json
{
    "Name": "Heavy Strike",
    "BaseDamage": 12,
    "MainStat": "STR",
    "RequiredStat": 8,
    "Scaling": 0.35,
    "DamageType": "Physical",
    "Tags": ["Melee"]
}
```

---

# Формула мастерства (Mastery)

Если характеристика ниже необходимой, карта работает слабее.

Формула:

```
Mastery = Clamp(
    CharacterStat / RequiredStat,
    0.30,
    1.00
)
```

Минимальная эффективность всегда составляет 30%.

Это означает:

* игрок не теряет возможность использовать карту;
* карта никогда не становится полностью бесполезной.

Пример.

Fireball:

```
BaseDamage = 6
RequiredStat = 8
```

Воин:

```
INT = 2

Mastery = 2 / 8 = 0.25

Clamp → 0.30
```

Итог:

```
Damage = 6 × 0.30 = 1.8

≈ 2
```

---

# Масштабирование (Scaling)

После достижения необходимой характеристики карта начинает усиливаться.

Формула:

```
Bonus =
max(
    0,
    CharacterStat - RequiredStat
) * Scaling
```

Итоговый урон:

```
Damage =
Round(
    BaseDamage * Mastery + Bonus
)
```

Пример.

Fireball

```
BaseDamage = 6
RequiredStat = 8
Scaling = 0.30
```

INT = 12

```
Mastery = 1

Bonus =
(12 - 8) × 0.30
=
1.2

Damage

=
7
```

---

# Общая формула

Каждая карта использует одинаковый алгоритм.

```
Mastery =
Clamp(
Stat / RequiredStat,
0.30,
1.00
)

Bonus =
max(
0,
Stat - RequiredStat
)
×
Scaling

Result =
Round(
BaseDamage × Mastery + Bonus
)
```

Это позволяет не писать индивидуальные формулы для каждой карты.

---

# Обычная атака

Персонаж всегда может атаковать без карты.

Базовая формула:

```
Damage =
BaseAttack
+
STR × 0.5
```

Например:

```
BaseAttack = 3
```

Воин:

```
STR = 10

Damage

=
8
```

Маг:

```
STR = 2

Damage

=
4
```

Таким образом STR имеет значение даже без использования карт.

# Назначение характеристик

## STR

Влияет на:

* физический урон;
* тяжелые атаки;
* пробивание брони;
* обычную атаку.

---

## AGI

Влияет на:

* шанс критического удара;
* уклонение;
* инициативу;
* дополнительные действия;
* снижение стоимости некоторых карт;
* вероятность повторной атаки.

---

## INT

Влияет на:

* магический урон;
* количество целей;
* силу заклинаний;
* мощность призыва.

---

## WIL

Влияет на:

* лечение;
* баффы;
* дебаффы;
* длительность эффектов;
* вероятность успешного контроля.

---

## VIT

Влияет на:

* максимальное здоровье;
* броню;
* сопротивление эффектам;
* регенерацию.

# Поддержка гибридных карт

Не все карты должны зависеть только от одной характеристики.

Система должна поддерживать несколько коэффициентов.

Пример.

```
Fireball

INT 80%

WIL 20%
```

Формула:

```
EffectiveStat =
INT × 0.8 +
WIL × 0.2
```

Другие примеры:

```
Heavy Strike

STR 80%
VIT 20%
```

```
Precise Shot

AGI 70%
INT 30%
```

```
Battle Cry

WIL 70%
STR 30%
```

Это позволяет создавать гибридные билды и делает развитие персонажа более вариативным.

---

# Принципы баланса

Система должна соблюдать следующие правила.

1. Любая карта доступна любому классу.

2. Недостаток характеристики снижает эффективность, но не запрещает использование.

3. Основная характеристика определяет силу карты.

4. После достижения требуемого значения характеристика продолжает усиливать карту.

5. У всех характеристик должна быть ценность независимо от класса.

6. Игрок должен иметь возможность создавать нестандартные и гибридные билды.

7. Все карты должны использовать единую систему расчета, чтобы баланс можно было настраивать изменением нескольких коэффициентов, а не переписыванием логики отдельных карт.
